Featured Project

Shift Manager

ケータリング現場向けの案件・勤務枠ベースのシフト管理アプリ

Shift Managerは、ケータリング現場のように 「案件ごとに勤務場所・勤務時間・必要人数が変わる」業務に対応するための シフト管理アプリです。

一般的な固定シフトではなく、案件、勤務枠、必要人数、従業員、割当を分離して管理することで、 現場の運用に近い形でシフトを扱えるようにすることを目指しました。

GitHub Repository Live Demo

Background

作成背景

このアプリは、ケータリング現場での実際の業務経験をもとに作成しました。

ケータリングの仕事では、毎回同じ店舗で同じ時間に働くわけではありません。 案件ごとに勤務場所、集合場所、勤務時間、必要人数、担当者が変わります。

そのため、一般的な店舗シフトのように 「曜日ごとに誰が入るか」を管理するだけでは不十分です。 現場では、案件ごとに何人必要で、現在何人が確定していて、 どの時間帯が不足しているのかを素早く把握する必要があります。

そこで、案件単位で勤務枠と割当状況を管理できるアプリとして Shift Managerを開発しました。

Problems & Solutions

解決したい課題

01

案件ごとに勤務条件が変わる

ケータリングでは案件ごとに場所・時間・必要人数が異なるため、 固定シフト型の管理では対応しにくい課題があります。

JobとJobShiftSlotを分離し、1つの案件に複数の勤務枠を持たせる設計にしました。
02

人員の充足状況が分かりにくい

必要人数に対して、現在何人が確定しているのかをすぐに把握できないと、 管理者の確認コストが大きくなります。

必要人数と割当人数を比較し、案件ごとの充足率を確認できるようにしました。
03

管理者とスタッフで必要な情報が違う

管理者は全体の割当状況を確認したい一方で、スタッフは自分の勤務予定を確認できれば十分です。

ロールに応じて管理画面とスタッフ画面を分ける設計を意識しました。

Design

設計上の工夫

現場の業務構造に合わせて、データモデルを分離しました。

Shift Managerでは、シフトを単なる予定表として扱うのではなく、 「案件」「勤務枠」「従業員」「割当」という単位に分けて管理しています。

Job 案件

案件名、日付、場所、メモなどを管理

↓
JobShiftSlot 勤務枠

勤務時間帯、必要人数を管理

↓
ShiftAssignment 割当

勤務枠に対して従業員を割り当てる

↓
Employee 従業員

名前、役割、勤務状態などを管理

このように責務を分けることで、1つの案件に複数の勤務時間帯がある場合や、 勤務枠ごとに必要人数が異なる場合にも対応しやすくなります。

Security

セキュリティ面の配慮

  • ログイン機能による未認証ユーザーのアクセス制限
  • 管理者・スタッフのロールに応じた画面分離
  • Server Actions実行時にも権限確認を行う設計
  • パスワードを平文保存せず、ハッシュ化して扱う設計
  • データベースURLや認証シークレットを環境変数で管理
  • 画面上の表示制御だけに依存しない認可処理

セキュリティについては、見た目上の表示制御だけでなく、 サーバー側でも認証・認可を確認することを重視しています。
