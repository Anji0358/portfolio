Featured Project

Shift Manager

Shift Manager Screenshot images/shift-manager.png を追加してください

ケータリング現場向けの案件・勤務枠ベースのシフト管理アプリ

ケータリング現場では、一般的な固定シフトとは異なり、 案件ごとに勤務場所・勤務時間・必要人数が変わります。 そのため、単純な「従業員 × 日付」のカレンダー型ではなく、 「案件」「勤務枠」「必要人数」「割当」を分けて管理できる仕組みが必要だと考えました。

Shift Manager では、自分自身のケータリング業務経験をもとに、 「どの案件に何人必要で、現在何人確定しているのか」を把握しやすくすることを目指しました。

Shift Manager Features

主な機能

案件管理

日付・勤務場所・案件名などを登録し、月ごとの案件一覧として管理できます。

勤務枠管理

1つの案件に対して、複数の勤務時間帯や必要人数を設定できるようにしています。

従業員管理

従業員情報、勤務状態、役割などを管理し、シフト割当に利用できます。

シフト割当

案件の勤務枠に対して従業員を割り当て、担当者を管理できます。

充足率表示

必要人数に対して現在何人が確定しているかを表示し、人員不足を把握しやすくします。

管理者向け画面

案件・従業員・割当状況を管理者が確認しやすいように整理しています。

Design

設計上の工夫

このアプリでは、シフトを単なる予定表として扱うのではなく、 ケータリング現場の業務構造に合わせてデータを分離しました。

Job 案件
↓
JobShiftSlot 勤務枠・必要人数
↓
ShiftAssignment 割当
↓
Employee 従業員

このように分けることで、1つの案件に複数の勤務時間帯がある場合や、 勤務枠ごとに必要人数が異なる場合にも対応できます。 また、必要人数と割当人数を比較することで、案件ごとの充足率を表示できるようにしました。

Learning

この開発で意識したこと

  • 業務構造を先に整理すること 画面から作り始めるのではなく、案件・勤務枠・従業員・割当というデータ構造を先に整理しました。
  • 現場の課題に合わせて設計すること 自分のケータリング業務経験をもとに、通常の店舗シフトとは異なる案件ベースの管理を意識しました。
  • 拡張しやすい構造にすること 今後、勤務不可情報、給与計算、出勤実績、管理者権限などを追加できるように、責務を分けて実装しています。
